Модель Habit (Привычка):
name – строка (например: "Утренняя зарядка", "Чтение книг")
category – строка, enum: ['health', 'education', 'productivity', 'mindfulness']
difficulty – строка, enum: ['easy', 'medium', 'hard']
streak – число (текущая серия дней), default: 0
bestStreak – число (рекордная серия), default: 0
totalCompletions – число (всего выполнений), default: 0
createdAt – дата

2. Модель Progress (Прогресс выполнения):
habitId – ObjectId, ref: 'Habit'
date – дата (день выполнения)
completed – булево
notes – строка (заметки о выполнении)
mood – число (оценка настроения от 1 до 5)

Базовые CRUD операции
Маршрут POST /habits — создаёт новую привычку. Принимает название, категорию, сложность.
Маршрут GET /habits — возвращает список всех привычек.
Маршрут GET /habits/:id — возвращает привычку по её идентификатору.
Маршрут PUT /habits/:id — обновляет данные привычки.
Маршрут DELETE /habits/:id — удаляет привычку.
*Маршрут POST /habits/:id/complete — отмечает выполнение привычки за сегодня. При этом должна автоматически обновляться статистика: если привычка выполнялась вчера, текущая серия (streak) увеличивается на 1. Если вчера был перерыв, серия сбрасывается до 1. Если текущая серия побила рекорд, обновляется bestStreak. Также увеличивается totalCompletions на 1.

Часть 2. Аналитические запросы
Вам нужно реализовать 8 аналитических запросов. Каждый запрос должен быть отдельным GET-маршрутом. **Для сложных вычислений используйте агрегации (aggregate).

Запрос 1. Серия побед (GET /stats/longest-streak)
Найдите привычку с самой длинной текущей активной серией выполнения. Верните название привычки, категорию, сложность и количество дней в текущей серии.
Пример ответа:
{
"success": true,
"message": "Привычка с самой длинной текущей серией",
"data": { "name": "Утренняя зарядка", "streak": 15, "category": "health", "difficulty": "medium" }
}

Запрос 2. Лучший день недели (GET /stats/best-day)

Определите, в какой день недели пользователь чаще всего выполняет привычки. Используйте агрегацию по дням недели. Верните день недели и количество выполнений.

Пример ответа:
{
"success": true,
"message": "Самый продуктивный день недели",
"data": { "dayName": "Понедельник", "count": 42 }
}

Запрос 3. Месяц-рекордсмен (GET /stats/best-month)

Посчитайте, сколько привычек выполнено в каждом месяце. Найдите месяц с максимальным количеством выполнений. Верните месяц, год и количество выполнений.

Пример ответа:
{
"success": true,
"message": "Самый продуктивный месяц",
"data": { "monthName": "Март", "year": 2024, "completions": 87 }
}

Запрос 4. Заброшенные привычки (GET /stats/abandoned)

Найдите привычки, которые не выполнялись более 7 дней. Верните список таких привычек с указанием названия, категории, даты последнего выполнения и количества дней с момента последнего выполнения.

Пример ответа:
{
"success": true,
"message": "Заброшенные привычки (не выполнялись более 7 дней)",
"count": 2,
"data": [
{ "name": "Чтение книг", "category": "education", "lastCompleted": "2024-01-15", "daysSince": 10 }
]
}

Запрос 5. Корреляция настроения (GET /stats/mood-correlation)

Вычислите среднее настроение для привычек разной сложности. Это покажет, как настроение влияет на выбор сложности привычки. Верните для каждой сложности (easy, medium, hard) средний балл настроения и количество выполнений.

Пример ответа:
{
"success": true,
"message": "Как настроение влияет на выбор сложности привычек",
"data": [
{ "difficulty": "easy", "averageMood": 4.2, "totalCompletions": 30 },
{ "difficulty": "medium", "averageMood": 3.8, "totalCompletions": 45 },
{ "difficulty": "hard", "averageMood": 3.5, "totalCompletions": 15 }
]
}

Запрос 6. Идеальный день (GET /stats/perfect-day)

Найдите день, когда было выполнено больше всего привычек и среднее настроение было выше 4. Верните дату, количество выполненных привычек и среднее настроение в этот день.

Пример ответа:
{
"success": true,
"message": "Идеальный день: максимум привычек и отличное настроение",
"data": { "date": "2024-03-15", "completions": 5, "averageMood": 4.6 }
}

Запрос 7. Золотая середина (GET /stats/golden-mean)

Найдите привычку, у которой количество выполнений (totalCompletions) находится ближе всего к среднему значению по всем привычкам. Верните название привычки, её количество выполнений и среднее значение по всем привычкам.

Пример ответа:
{
"success": true,
"message": "Привычка, ближе всего к среднему количеству выполнений",
"data": {
"habit": { "name": "Медитация", "totalCompletions": 42 },
"averageCompletions": 45.3,
"difference": 3.3
}
}

Запрос 8. Выгорание (GET /stats/burnout)

Найдите привычки, у которых текущая серия (streak) больше, чем общее количество выполнений (totalCompletions). Это может означать, что пользователь начал пропускать дни после длинной серии. Верните название привычки, текущую серию, общее количество выполнений и разницу.

Пример ответа:
{
"success": true,
"message": "Привычки на грани выгорания",
"count": 1,
"data": [
{ "name": "Английский язык", "streak": 50, "totalCompletions": 48, "difference": 2 }
]
}
