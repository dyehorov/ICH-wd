
Создайте HTTP сервер, который копирует большие файлы с использованием потоков (streams) и генерирует события о процессе копирования. Все действия определяются по URL (без GET-параметров).
Требования:
Создайте проект streamCopyServer и файл server.js
Импортируйте модули http, fs, path, events
Создайте экземпляр EventEmitter
Зарегистрируйте обработчики событий:
copyStart – выводит в консоль "[НАЧАЛО] Копирование файла начато"
copyProgress – выводит в консоль "[ПРОГРЕСС] Скопировано порция данных"
copyComplete – выводит в консоль "[ЗАВЕРШЕНО] Копирование успешно завершено"
copyError – выводит в консоль "[ОШИБКА]"
Создайте HTTP сервер со следующими маршрутами (все без параметров, действия определяются по URL):
Маршрут "/copy/small" – копирует файл small.txt в small_copy.txt с использованием потоков. Генерирует события copyStart, copyProgress, copyComplete или copyError
Маршрут "/copy/large" – копирует файл large.txt в large_copy.txt с использованием потоков. Генерирует события copyStart, copyProgress (каждые 64KB), copyComplete или copyError
Маршрут "/copy/image" – копирует файл photo.jpg в photo_copy.jpg с использованием потоков. Генерирует события copyStart, copyProgress, copyComplete или copyError
Маршрут "/copy/backup" – копирует файл data.txt в backup/data.txt (создавая каталог backup при необходимости). Генерирует соответствующие события
При копировании обработайте событие data на readStream для генерации события copyProgress каждые 64KB
Если исходный файл не существует, сервер должен вернуть ошибку 404 и сгенерировать событие copyError
Запустите сервер на порту 3333
